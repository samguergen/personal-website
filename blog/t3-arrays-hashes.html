Ruby, Arrays & Hashes.

This week, I immersed myself in Ruby and built some simple programs using the concepts we learned about control flow, loops, iterators, arrays and hashes. Control flow comes quite naturally for me, since it derives from logic, my favorite class as an undergraduate. The if else statements and while loops form the core of computer programming: a two-fold (or binary) mechanism that gives the computer its intelligence. I call it the core, because most computer programs' foundations are built on binary thinking style, which we can more or less interpret as:
Either something is this, or something is not.
Using this pattern as a building block, the program will then understand more complex logic.

 

Arrays

Arrays and hashes allows data to be assembled in an organized way, thematically or not, and boxed into one variable for further use.
For example, let's say I were to create a variable for all of my favorite movies, so that later I could create a program that lists one, many or all of them as part of a method for an announcer's speech.
Without arrays, I would need to create a variable for each of them, and remember the name of each variable I called to later call them in my method. With an array, I would simply store all of the movies names into one variable, and later, in my method, if I wish to print all of the movie names, I could do so by concatenatening the annoucement string with the array name.

For example:

 

#I build my array

favorites_movies = ["The Matrix", "Avatar", "The Lion King", "The Notebook"]

 

# I build my announcer method, with a placeholder as parameter

def announcer(movies)
puts "Hi, and thank you for attending the Annual Sam Movie Night!"
puts "Tonight, we will be presenting: " + movies.join(", ") + "!"
end

 

#I call my method, with my favorite_movies array as parameter

announcer(favorite_movies)
=>
Hi, and thank you for attending the Annual Sam Movie Night!
Tonight, we will be presenting: The Matrix, Avatar, The Lion King, The Notebook!

 

Without arrays, I have to add each movie as an input, which is considerably more tedious:

announcer("The Matrix, Avatar, The Lion King, The Notebook")
=>
Hi, and thank you for attending the Annual Sam Movie Night!
Tonight, we will be presenting: The Matrix, Avatar, The Lion King, The Notebook!

 

Alternatively, if I wish to only announce one or a few movies, I can print them out using the array name and their position in the array:

announcer(favorite_movies[0:2])
=>
Hi, and thank you for attending the Annual Sam Movie Night!
Tonight, we will be presenting: The Matrix, Avatar, The Lion King!

 

Hashes

Similar to an array, a hash stores a list of data into one variable. However, the difference is that instead of individual data, with a default index, a hash stores a list of key-value pairs.
Using my previous example, if I were to convert my array into a hash, I would instead place a value for each movie name. For example, my opinion of the movie:

favorites_movies = ["The Matrix": "deep, obscure, and thought-provoking",
"Avatar": "futurist and dreamy",
"The Lion King": "childhood classic",
"The Notebook": "chick flick"]

 

Consequently, I were to call the an element according to its position in the hash, I would first need to conver the hash into an array, because hashes have no default index. In doing so, the key-value pair would be returned, not the individual value:

favorite_movies.to_a[1]
=>
Avatar: futurist and dreamy

 

If I were to return the value instead (my opinion of a movie), I would call it by its key, not by its default index.

favorite_movies["Avatar"]
(or sorting by index, but specifying amout the values only)
favorites_movies.values[1]
=>
futurist and dreamy

 

Thus, a key can be considered the index of its associated value.
