SQL Basics

Introduction & Basics
    What is SQL?
  • DBMS = database management system
  • Give you info between you and your data
  • SQL= language to query database
  • The point of SQL is to change datasets (not single data, but entire rows or columns)
    Structure
  • Database is like a table
  • Every column:
  • stored a piece of data.
  • Has unique data type
  • Can be formatted to only accept certain data type or structure (like phone)

  • Every row stores value for that data;
  • Separate instance of the data.
  • Such as a contact person.
    Constraints
  • minimum 1 row per database
  • 2 rows can’t contain same info
  • column should have only 1 datatype
  • for e.g:
    column should be number only in format 000-000-0000
    only picture can be updated in row, not name
Relational Databases
    What is it?
  • Relational database is relations of rows together
  • Order of rows of columns doesn’t matter.
  • ID is useful because every instance (row) may have same name, same country, etc.
  • You can create new table base on different tables, like enrolment for student and class
  • The power of relational database lies in the relationship that can be defined between tables.
  • The most crucial aspect in designing a relational database is to identify the relationships among tables:
    One-to-Many Relationship
  • for e.g: 1 car has many seats. But the seats only belong to the car, not to another.
  • Table A has its own key + foreign key from Table B. Table B has its own key (linking to Table A's foreign key). But no foreign key of its own.
    One-to-One Relationship
  • for e.g: countries and cities
  • any given row from Table A can have at most one matching row in Table B.
  • child (cities) does not have its own id, because can’t exist without parent- only has id of parent
    Many-to-Many Relationship
  • for e.g: book and authors
  • rows in the first table can map to multiple rows in the second table… and those rows in the second table can also map to multiple (different) rows in the first table.
  • requires join table, with foreign keys (id) of both
  • by convention, join table has name of both (name1_name2) and no id of its own.
