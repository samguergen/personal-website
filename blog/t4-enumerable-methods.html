Enumerables

This week, I learned about enumerable modules. These are quite useful and typical of Ruby, because they allow us to iterate (or loop) through an array or hash, in order to gather or modify every element in an array.

So what makes an enumerable different than a for or while loop?

Enumerables can provide a tailored way of iterating through the array or hash, which is implicit and most often characterized by its name. For example, the .sort method iterates through an array, and in doing so it will sort the elements alphabetically or numerically:

students= ["Sandra","Kevin","Sam", "Pat","Catherine","Amandine","Lucia"]

#will sort the names alphabetically

students.sort
=>
["Amandine", "Catherine", "Kevin", "Lucia", "Pat", "Sam", "Sandra"]

This provides a great advantage for Rubyists: it saves them lots of lines of code!
Here is why: technically, you could manually define a sort method using only for and if loops:

But, we'll have to admit that using .sort is just much more convenient and easy.

 

numerables allow us to write even more complex logic in fewer lines, because we can customize these already-customized loops by our own criteria, that we define inside its associated block.

The general format goes:

array.enumerable{|i| customized criteria involving i}

For example:

students= ["Sandra","Kevin","Sam","Pat","Catherine","Amandine","Lucia"]

#will sort names by number of characters in name

students.sort_by {|name| name.length}
=>
["Lucia", "Pat", "Sandra", "Catherine", "Sam", "Amandine", "Kevin"]

 

The enumerable method that I used most often this week was .map{block}. This method is similar to the standard .each method (what I like to call a 'naked' enumerator, since it has no implicit customization and works the same as a for loop) but instead, produces a new array with the result of calling the block. .Each, on the other hand, evaluates the block and throws away the result afterwards.

Another method I used a lot was group_by{block}, which iterates through an array and groups the element in the way specified by the block. The output returns in hash format, with the grouping category as key, and the elements grouped in that category as values for that key.

For example:

students= ["Sandra","Kevin","Sam","Pat","Catherine","Amandine","Lucia"]

students.group_by{|name| name.length}
=>
{5=>["Kevin", "Lucia"], 6=>["Sandra"], 8=>["Amandine"], 3=>["Sam", "Pat"], 9=>["Catherine"]}

The hash contains keys, which correspond to the different name lengths, and the values, which correspond to every name with that particular length.

Enumerables seem to be the most complex part of my Ruby learning so far. Everything revolving around looping requires careful logic, but since enumerables are ready-made customized loops, requiring in most cases even more customizing (what you define inside the block) it can be easy to lose track of how you are iterating through you array or hash. Enumerables can be both fun and save you tons of lines of coding, but handle them with care!
